# ADR-002: Chunking & Metadata Strategy for Hierarchical Medical Documents

## Status
Accepted (2026-05-31)

## Context
Medical documents like the Merck Manual have strong hierarchical structure (Book → Section → Chapter → Subsection → Tables → Black-box warnings). The previous PoC used naive recursive chunking, which often breaks context and makes accurate citation difficult. We need a strategy that maximizes grounding, citation fidelity, and medical safety.

## Decision

**1. Primary Chunking Strategy**
Adopt **HierarchicalMedicalChunker** as the production strategy because it respects document structure and preserves semantic boundaries critical for clinical accuracy.

Implement **RecursiveMedicalChunker** (baseline) for comparison and educational purposes. The ingestion pipeline will be modular (Strategy Pattern) so its easy to switch and benchmark both.

**2. Metadata Schema** (every chunk must contain)
- `source_title`
- `section_hierarchy` (list: ["Book", "Cardiology", "Heart Failure"])
- `page_number`
- `chunk_type` ("section", "subsection", "table", "warning")
- `chunk_id` (deterministic)
- `parent_chunk_id` (for hierarchy navigation)

**3. Citation Strategy**
- Always return `Source: [Title] - Section: [hierarchy] - Page [X]`
- Use chunk metadata to build rich citations in the final response
- Safety layer will verify presence of citation

**4. Chunk Size & Overlap Policy** (Medical-tuned)
- Base: 800-1200 tokens
- Overlap: 200 tokens (higher than usual to preserve context across subsections)
- Special handling for tables and black-box warnings (smaller, higher priority chunks)

**5. Extensibility & Modularity**
- Abstract `BaseChunkingStrategy`
- Concrete classes: `RecursiveMedicalChunker`, `HierarchicalMedicalChunker`
- Factory pattern in ingestion layer
- Easy to add SemanticChunker or Agentic chunking later

## Alternatives Considered
- Pure RecursiveCharacterTextSplitter → Rejected (loses hierarchy and citation quality)
- Pure SemanticChunker → Too slow and unpredictable for medical safety

## Consequences
- Positive: Much higher grounding accuracy, better citations, strong educational value (we can compare both chunkers), excellent portfolio signal.
- Negative: Slightly more complex ingestion code (mitigated by clean design).
- Long-term: This design scales to multi-document RAG and evaluation harness naturally.
