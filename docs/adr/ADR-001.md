# ADR-001: Clean Architecture + Local-First Stack with Grok API and FAISS

## Status
Accepted (2026-05-26)

## Context
Building production-grade privacy-first RAG for medical use. Previous notebook was flat PoC. Need maintainable, testable structure from day one.

## Decision
- Clean architecture with `src/frontline_clinical_rag/` packages
- Embeddings: BAAI/bge-m3 (local)
- Vector store: FAISS (local)
- LLM: Grok via xAI REST API
- Framework: LangChain
- Config: env.example + .env

## Alternatives Considered
- ChromaDB → rejected for lighter FAISS performance
- Fully local Ollama → rejected for stronger REST API portfolio signal
- Flat structure → rejected for senior engineering standards

## Consequences
Positive: Fast retrieval, easy testing, strong portfolio signal.  
Negative: Minor index management (handled by LangChain).  
Long-term: Scales cleanly to agents and production.
